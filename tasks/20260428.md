## Recommended next moves

1. **Don't deploy signed_v2 to live yet.** The 04-27 test PnL is one day. Run a paper session against v2 for 2–3 days first. If it holds up, swap in.
2. **Wire `recent_slot_outcomes` into the live cycle_runner** — currently the S3 loader populates it (so training works) but the live bot still passes nothing, so live predictions get the 0.5 prior on those features. The model trained on real outcome rates expects the live signal too.
3. **Run `feature_probe.py` on `experiments/signed-v2/dataset.parquet`** to know which of the 52 active features actually carry weight in the new regime. Some of the v1 survivors may now be redundant given the new regime features.
4. The `ut_trend_disagreement` feature fires on **40% of slots** — that's a lot of regime context. If feature_probe ranks it high, this validates the "free win" hypothesis from the design.
