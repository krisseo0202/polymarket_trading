# Journal: 2026-04-11

## Model analysis & v5 training

Started with a model audit notebook (`notebooks/model_analysis.ipynb`) for logreg_v4. Found the sklearn pickle was broken — `predict_proba` crashed on `multi_class` attribute removed in newer sklearn. Fixed by computing `sigmoid(X @ coef + intercept)` directly from the fitted params. No sklearn version downgrade needed.

Trained logreg_v5 with a new 13-feature compact microstructure set:
`ret_30s, ret_60s, depth_ratio, depth_skew, imbalance, spread, wall_flag, depth_change, mid_return_5s, acceleration, rolling_std_30s, range_30s, time_to_expiry`

v5 traded better in backtest ($313 vs v4's $291) but had worse calibration (Brier 0.2178 vs 0.2070). The features capture Polymarket orderbook dynamics that v4 missed, but the model isn't confident enough in the right moments.

## Optuna + LLM tuning loop

Built a closed-loop hyperparameter optimization system:

1. **`tune_v5_optuna.py`** — multi-objective Optuna (minimize Brier, maximize backtest equity). NSGA-II sampler. SQLite storage per round.
2. **`llm_suggest_v5.py`** — after each Optuna round, Claude Sonnet 4.6 reads the Pareto front and proposes the next round's feature subset + tightened search space + seed trial.
3. **`tune_and_suggest_loop.py`** — orchestrates N rounds: Optuna -> LLM -> next Optuna seeded with LLM's config.

Ran 3 rounds x 100 trials. LLM's key moves:
- Round 0 -> 1: dropped `depth_skew`, `ret_60s`, `range_30s` as collinear; imported `vol_ratio_15_60` and `cvd_60s` from v4 pool; tightened C range.
- Round 1 -> 2: shifted C upward to [0.02, 0.15] after noticing low-C trials had worst Brier; added `vwap_deviation`.

### LLM-assisted vs pure Optuna (300 trials each)

| Metric            | LLM track | Pure Optuna |
|-------------------|-----------|-------------|
| Best Brier        | 0.2105    | 0.2162      |
| Best equity       | $3,657    | $1,646      |
| Pareto size       | 8         | 4           |
| Hypervolume       | 2,800     | 1,212       |

LLM won on every axis. The pure Optuna control kept exploiting the same basin (C~0.008, row_interval=20) and never found the C~0.1 calibration sweet spot that the LLM pushed toward.

## v5 tuned vs v4 comparison

Persisted round-2 Pareto trial #40 as `models/logreg_v5_tuned/`:
- 14 features, C=0.1186, row_interval=25, edge_threshold=0.049, kelly_mult=0.425, max_frac=0.247

| Metric    | v4       | v5_tuned |
|-----------|----------|----------|
| Brier     | **0.2074** | 0.2106 |
| Slot acc  | **0.7077** | 0.6308 |

v4 is still the better probability estimator. v5_tuned's PnL win comes largely from aggressive sizing (kelly_mult=0.425, max_frac=24.7%) that's likely overfit to the 65-slot held-out tail. For live trading, v4 with conservative sizing remains the safer choice.

## Key takeaways

1. The LLM experiment suggester works. It finds search regions that NSGA-II alone wouldn't explore, because it reasons about *why* certain trials failed (e.g. "low C = bad calibration") and shifts the space accordingly.
2. Calibration (Brier) and PnL can point in different directions. The tuner optimizes both, but the PnL objective is noisy on small held-out sets. Trust Brier for live decisions.
3. The v5 microstructure features have signal but the model isn't expressive enough to use them well — isotonic calibrator quantizes 79 raw scores into 6 outputs in live trading.
