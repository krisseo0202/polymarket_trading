# Implementation Plan: End-to-End BTC Probability Engine + Cycle-Aligned Execution

**Branch**: `001-btc-prob-cycle-engine` | **Date**: 2026-03-18 | **Spec**: [spec.md](./spec.md)

**Delivery strategy**: Minimal working pipeline first ‚Üí harden ‚Üí then sophisticate.
Do not over-engineer the first pass. Each phase must be independently testable and
leave the repo in a working state.

---

## Summary

Wire six existing utility components (BTC price feed, vol estimator, GBM Monte Carlo,
Polymarket odds fetcher, cycle scheduler, Kelly sizer) into a coherent execution pipeline.
The only new *production* code is a discrete `EdgeDetector` component (currently missing)
and a thin `pipeline.py` wiring module. All other work is tests and hardening.

The implementation follows a **Minimal ‚Üí Medium ‚Üí Sophisticated** track. This plan
covers Minimal (Phase 1‚Äì3) in full. Medium and Sophisticated are outlined for reference.

---

## Technical Context

**Language/Version**: Python 3.10+
**Primary Dependencies**: `numpy` (MC simulation), `websockets` (BTC feed), `requests` (Polymarket API), `pytest` + `unittest.mock` (tests)
**Storage**: None (in-memory; no DB required)
**Testing**: `pytest` ‚Äî existing suite in `tests/utils/`
**Target Platform**: macOS/Linux; runs as a long-lived daemon thread inside the trading bot
**Project Type**: Library + integration harness (no web service, no CLI)
**Performance Goals**: Full probability estimate (vol + MC 1k paths) < 500ms; trigger jitter < 1s
**Constraints**: Tests must run with zero live network calls (all I/O mocked); dry-run mode required
**Scale/Scope**: Single-market, single-asset pipeline; one execution per 5-minute cycle

## Phase 1 ‚Äî Minimal: Tests for Existing Components

**Goal**: Establish test coverage for the four untested components before touching any
production code. Each test file must be independently runnable.

### 1.1 ‚Äî BTC Feed Tests (`tests/utils/test_btc_feed.py`)

Test `BtcPriceFeed` in isolation using a fake WebSocket message injector.

**Key cases**:
- Feed initialises with `is_healthy() == False` before first message
- After injecting a bookTicker message, `get_latest_mid()` returns correct mid, `is_healthy()` ‚Üí True
- After injecting no messages for `stale_warn_s` seconds, `is_healthy()` ‚Üí False
- `get_recent_prices(n)` returns only messages within the time window
- Malformed JSON / missing bid-ask fields are skipped silently

**Mock strategy**: Inject messages directly via `feed._handle_message(raw_json)` ‚Äî no real WebSocket needed.

### 1.2 ‚Äî Cycle Scheduler Tests (`tests/utils/test_cycle_scheduler.py`)

Test `detect_current_cycle`, `cycle_index`, `aligned_cycle_anchor`, and `run_last_second_strategy`.

**Key cases**:
- `detect_current_cycle(anchor, 10)` returns correct seconds-remaining for known elapsed times
- `cycle_index` increments at the right boundary
- `run_last_second_strategy` fires callback exactly once per cycle in a short-cycle (1s cycle, 0.2s trigger)
- Callback exception does not crash the loop; next cycle fires normally
- Stop event terminates the loop within 1 poll interval

**Mock strategy**: Use short cycles (1‚Äì2 seconds) in tests so they run fast. No mocking needed ‚Äî pure time math.

### 1.3 ‚Äî Probability Tests (`tests/utils/test_probability.py`)

Test `simulate_up_prob` from `quant_desk.py`.

**Key cases**:
- `simulate_up_prob(start=100, current=100, time_left=60, vol=0.5)` returns value in [0.45, 0.55]
- `simulate_up_prob(start=100, current=200, time_left=1, vol=0.001)` returns value > 0.99
- `simulate_up_prob(start=100, current=50, time_left=1, vol=0.001)` returns value < 0.01
- Does not raise for `time_left_sec=0.001` (near-zero time)
- Result is a float, not a numpy scalar

### 1.4 ‚Äî Market Utils Tests (`tests/utils/test_market_utils.py`)

Test `fetch_market_odds` with mocked HTTP responses.

**Key cases**:
- Valid response: returns `(up_prob, down_prob)` both in [0, 1]
- Market not found (empty list): raises `ValueError`
- Missing clobTokenIds: raises `ValueError`
- Network timeout: retries N times then raises `RuntimeError`
- HTTP 500: retries (transient); HTTP 404: raises immediately (not retried)

**Mock strategy**: `unittest.mock.patch("requests.get")` returning pre-built response objects.

## Phase 2 ‚Äî Minimal: Edge Detector (New Component)

**Goal**: Create the one missing production component ‚Äî the edge detection function.
This is a pure function with no I/O; it is completely determined by its inputs.

### 2.1 ‚Äî `src/utils/edge_detector.py`

```python
# Public interface (not implementation ‚Äî just the contract)

@dataclass
class EdgeDecision:
    side: str              # "BUY_UP" | "BUY_DOWN" | "NO_TRADE"
    bot_up_prob: float
    market_up_odds: float
    market_down_odds: float
    up_edge: float
    down_edge: float
    edge: float            # edge on the selected side
    kelly_fraction: float
    min_edge_threshold: float

def detect_edge(
    bot_up_prob: float,
    market_up_odds: float,
    market_down_odds: float,
    min_edge: float = 0.03,
    max_kelly: float = 0.05,
) -> EdgeDecision:
    ...
```

**Edge logic**:
- `up_edge = bot_up_prob - market_up_odds`
- `down_edge = (1 - bot_up_prob) - market_down_odds`
- If both exceed `min_edge`: pick side with larger edge
- If one exceeds `min_edge`: pick that side
- Otherwise: `NO_TRADE`

**Kelly calculation**: uses `kelly_fraction(p, q, odds)` from `src/utils/kelly.py`.
For UP: `p=bot_up_prob`, `q=1-bot_up_prob`, `odds = (1 / market_up_odds) - 1`

### 2.2 ‚Äî `tests/utils/test_edge_detector.py`

All test cases are parameterized, pure-function calls ‚Äî no mocking needed.

**Key cases**:
- UP edge detected when `bot_up_prob - market_up_odds >= min_edge`
- DOWN edge detected when `(1-bot_up_prob) - market_down_odds >= min_edge`
- NO_TRADE when both edges below threshold
- Larger edge wins when both sides positive
- Kelly fraction is 0 for NO_TRADE
- Kelly fraction is capped at `max_kelly` for large edges
- Output values are within valid ranges

### 2.3 ‚Äî Update `src/utils/__init__.py`

Export `EdgeDecision` and `detect_edge` from the utils package.

## Phase 3 ‚Äî Minimal: Pipeline Integration

**Goal**: Wire all components into a single runnable module with dry-run support and
a full integration test.

### 3.1 ‚Äî `src/pipeline.py`

**Public interface**:

```python
@dataclass
class PipelineConfig:
    market_id: str
    cycle_len: int = 300
    trigger_window_s: float = 30.0
    vol_window_s: float = 300.0
    vol_method: str = "ema"
    n_paths: int = 1000
    min_edge: float = 0.03
    max_kelly: float = 0.05
    dry_run: bool = True
    btc_symbol: str = "btcusdt"

def run_pipeline(
    config: PipelineConfig,
    client=None,           # PolymarketClient; None = always dry-run
    stop_event=None,       # threading.Event; None = run forever
    logger=None,
) -> None:
    """
    Start the BTC price feed and cycle scheduler. Blocks until stop_event is set.
    Each cycle fires the probability ‚Üí edge ‚Üí (optional) order callback.
    """
```

**Internal logic**:
1. Start `BtcPriceFeed`
2. Compute `anchor = aligned_cycle_anchor(config.cycle_len)`
3. Maintain `start_price: Optional[float]` ‚Äî updated at each cycle rollover via `cycle_index` tracking
4. Callback (fired by `run_last_second_strategy`):
   a. Check feed health ‚Üí skip if unhealthy (log + `CycleTick(skipped=True)`)
   b. `prices = feed.get_recent_prices(config.vol_window_s)` ‚Üí extract mid list
   c. `vol = estimate_realized_vol([m for _, m in prices], config.vol_window_s, config.vol_method)`
   d. `up_prob = simulate_up_prob(start_price, feed.get_latest_mid(), remaining_s, vol)`
   e. `up_odds, down_odds = fetch_market_odds(config.market_id)`
   f. `decision = detect_edge(up_prob, up_odds, down_odds, config.min_edge, config.max_kelly)`
   g. Log `CycleTick`
   h. If not dry_run and client and decision.side != "NO_TRADE" ‚Üí place order

### 3.2 ‚Äî `tests/integration/test_pipeline.py`

Full pipeline test with all external I/O mocked.

**Setup**:
- Patch `BtcPriceFeed` to inject fake prices on demand
- Patch `fetch_market_odds` to return `(0.60, 0.40)`
- Use `cycle_len=2, trigger_window_s=0.5` for a 2-second test cycle
- Run for 2 cycles (4 seconds total)

**Assertions**:
- Callback fired exactly 2 times
- Each `CycleTick` has valid `up_probability`, `edge_decision`, `feed_healthy=True`
- In dry-run mode: client.place_order never called
- Stop event terminates the pipeline within 1 second

## Phase 4 (Medium ‚Äî Future)

*Not part of this implementation. Recorded for roadmap awareness.*

- Replace MC probability with Black-Scholes closed form (`p = N(d2)`) ‚Äî removes MC variance, sub-ms runtime
- Add short-window vol (30s) alongside long-window vol; use `max(vol_short, vol_long)` for risk-off
- Cache market odds between cycles (HTTP call moved out of tight callback)
- Add a circuit breaker: skip 3 consecutive cycles if edge consistently near zero
- Persist `CycleTick` history to a rolling log file for post-session analysis

## Phase 5 (Sophisticated ‚Äî Future)

*Not part of this implementation.*

- Jump-diffusion model for BTC (Merton 1976) to account for gap risk
- Order-book imbalance as drift signal: `mu = alpha * (bid_depth - ask_depth) / (bid_depth + ask_depth)`
- Multi-market correlation: use BTC volatility as a signal for ETH Up/Down markets
- Full async pipeline (asyncio) for sub-100ms end-to-end latency
